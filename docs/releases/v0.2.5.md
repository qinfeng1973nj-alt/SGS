# v0.2.5 — Error Handling Hardening

## Summary
This release hardens error handling by unifying all non-200 HTTP responses into the `ErrorEnvelope` contract.

## Changes
- Added global handler for `StarletteHTTPException` in `app/main.py`.
- Kept dedicated adapter for `RequestValidationError` to preserve `/score` compatibility.
- Ensured 404/405 and validation errors return structured `ErrorEnvelope` instead of default `{"detail": ...}`.
- Added contract tests in `tests/test_error_envelope_http.py`:
  - unknown route -> 404 envelope
  - `/score` missing required field -> 400 envelope
- Added `tests/conftest.py` with shared `client` fixture for test setup consistency.
