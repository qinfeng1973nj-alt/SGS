# v0.2.7 Trace Cleanup

## Scope
- normalize incoming trace header priority: X-Trace-Id > X-Request-ID
- keep response header `X-Trace-Id` always present
- ensure error envelope field `error.trace_id` equals response header
- add regression tests for missing/empty/whitespace trace header cases

## Non-Goals
- no API contract breaking changes
- no business logic changes outside trace flow

## Acceptance Criteria
1. request without trace headers => server generates trace id
2. request with `X-Trace-Id` => same id returned in response header
3. request with only `X-Request-ID` => mapped to `X-Trace-Id`
4. empty/blank trace headers are treated as missing and regenerated
5. non-2xx responses keep `ok: false` envelope and correct `error.trace_id`
